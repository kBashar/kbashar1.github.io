"My GSOC at PDFBox"

As a gsoc student I'm going to code to improve the PDFDebugger tool of the PDFBOX this summer. PDFBox is from the apache foundation.

PDFDebugger helps you to inspect the internal data in a structured and readable way which in turn paves a way to debug your PDF files. The PDFBOX already has a PDFDebugger which provides some fundamental facilities to user but lacks many more which are very useful for PDF detective works. My project is to improve this tool and make it useable and friendlier for the users as much as possible. The features I'm going to add are from the PDFBOX-2530 jira issue. If time permits, I may code some more.

My mentor is Tilman Hausherr. I contacted with him in mid of the November of 2014 and from the beginning he was very helpful to get me on the track. Really lucky to have someone like him right beside me as a guide in the journey of learning, exploring and doing. I’m hopeful to spent a fun summer while completing the GSOC as success story.

Operator precedence in java

I was just trying to solve this warmup problem in HackerRank. I was confident about getting it right by the first try. I was wrong. I was writing the expression wrong in first look which wasn't so vivid.

int k = 5;
System.out.println((k%2==0)? (k/2)*(k/2):((k/2)+1)*k/2);

prints 7 but I was expecting 6. Well, I think I should explain what is expected here. If k is even output will be square of k/2 (as in integer division) while in case of odd output is k/2 is multiplied with k/2 + 1. That is for 5 output calculated like 5/2 = 2 and then 2*(2+1) = 6. So when I realized there may be something wrong the precedence I wrote like below which gave expected the result.

System.out.println((k%2==0)? (k/2)*(k/2):((k/2)+1)*(k/2));

But it's a solution using parentheses and to get the understanding what's going on here I tried it like below

System.out.println((k%2==0)? (k/2)*(k/2):k/2*((k/2)+1));

and again I got the expected result. So it's clear that I've wrong understanding in operator precedence in Java and hit the google to get the gist. The second link from Princeton was a bookmark worth link, a crystal-clear explanation on operator precedence. In Java to evaluate an expression first operator precedence is looked then the associativity. So in my first expression after evaluating (5/2)+1 expression stands like 3*5/2. Now * and / both have same precedence and evaluation works using associativity. Both * and / are left associative that is 3*5/2 is (3*5)/2 that is 15/2 and the result is 7. In the third case after evaluating (5/2)+1 expression stands like 5/2*3 and for the sake of associativity it's (5/2)*3 and the result is 2*3 = 6.

The Princeton article has some interesting exercises in the end and they are way more interesting. I'm giving an example.

System.out.println("1 + 2 = " + 1 + 2);
System.out.println("1 + 2 = " + (1 + 2));

First expression prints 1 + 2 = 12 and

second one prints 1 + 2 = 3. For the explanation, I would recommend you to read the article
