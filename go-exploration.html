Only "for" loop is real

@6:05PM

There is only one loop expression in Go. That is for loop. As other languages loop expression construct is same i.e. there is an init statement, condition expression and post statement. but unlike other languages like C, Java, or Javascript there are no parentheses surrounding the three components of the for statement and the braces { } are always required. As usual for loop for i := 0; i<10; i++{ sum +=i } Without initial and post statement

        for ; i<10; {
            sum +=i
        }

We can choose to not use those semicolons there and now it looks like while loop. voila! for i<10{ sum +=i } And there is a forever loop also and like others it can be achived via for loop. Omitting conditional expression and init and post statement turns a for loop into forever loop. 

        for {
                sum +=i
        }

If statement

@6:30PM

If statements like for loop don't have any parentheses surrounding the condition statement. Short statement can be placed before condition statement and variables have scope inside if & else blocks. func pow(x, n, lim float64) float64 { if v := math.Pow(x, n); v < lim { return v } else { fmt.Printf("%g >= %g\n", v, lim) } // can't use v here, though return lim } I've just noticied math pacakage accepts mostly float64 type arguments while using pow function from math package.

@7:10 Newton-rhapson's method for finding sqrt

In go tutorial there is an excercise for finding sqrt of numbers for better understanding of loop and functions in go. I knew about the newton method but to brush up the idea I've gone throw this article from MIT. https://mitpress.mit.edu/sicp/chapter1/node9.html This is my implementation. Well, coding was really smooth. I didn't feel like I'm codig a new langauge.

func Sqrt(x float64) float64 {
    z := float64(1)
    for z>0{
        new_z := (x/z + z)/2
        fmt.Println(z, new_z)
        if v := math.Abs(z-new_z); v<0.00001{
            return new_z
        }
        z = new_z
    }
    return z
}
